Chapter 1
Introduction

1.1 What is Icecast?

  • Icecast is a streaming media server which currently supports Ogg Vorbis/Theora, Opus, WebM and MPEG Audio Layer 3 (MP3) streams.

1.2 Licensing

  • Icecast is distributed under the GNU GPL, version 2, which basically means:
    1. You can use the source code to develop new open source GNU GPL V2 applications and
    2. you can NOT use the source code to develop propietary applications.

Chapter 2
Architecture

  • Source Clients: Send the content to Icecast servers.
  • Servers: Sends content to Listeners and Relay Servers.
  • Listeners: Plays the content.
  • Relay servers: Act as listeners to servers, but are Servers.

Chapter 3
Setup

3.1 The conﬁguration ﬁle

  • Edit the /etc/icecast2/icecast.xml ﬁle and set:
    1. The <source-password>, which will be used for the source client authentication,
    2. <admin-password> that will be used for authenticating admin features of icecast.
  • By default, Icecast2 sends data to listeners throught the port 8000.

3.2 Testing possible /etc/icecast2/icecast.xml syntax problems

  sudo xmllint /etc/icecast2/icecast.xml
  • If you get only the content of /etc/icecast2/icecast.xml, the ﬁle is OK.

Chapter 4
Usage

4.1 Installation

# Debians:  
sudo apt-get install icecast2 # Configure it!  
 
# The server is running now and scheduled to run after rebotting the host.  
# However, if this does not work, it can be launched manually with:  
sudo icecast2 -c /etc/icecast2/icecast.xml  
 
# Or even better with:  
sudo /etc/init.d/icecast2 restart

4.2 Checking the server status

  • Options:
    1. Check the error.log ﬁle for the ’server started’ message. It should include something like:
      head /var/log/icecast2/error.log  
      #[2003-10-31  13:04:49] INFO main/main.c Icecast 2.3.0 server started

    2. By visiting the main page:
      firefox http://serverIP:port/admin/stats.xml &

    3. Telneting the serverIP:port.
      telnet localhost 8000

4.3 Conﬁguring a Source Client

  • A Source Client provides content to a Server or Relay Server.
  • It is neccesary to:
    1. Specify the IP address and the port of the icecast server.
    2. Choose a mountpoint. Example:
      # oggfwd server_host server_port source_pass mountpoint < file.ogg  
      oggfwd localhost 8000 hackme /ejemplo.ogg < Big_Buck_Bunny_small.ogv

  • Verify that it is connected by retrieving the stats.xml ﬁle from Icecast.

4.4 Conﬁguring a Listener

  • A Listener usually plays the content.
  • It is necessary to specify a .m3u or directly the mountpoint. Examples:
    vlc http://localhost:8000/mystream.ogg.m3u  
    vlc http://localhost:8000/mystream.ogg  
    wget http://localhost:8000/mystream.ogg  
    firefox http://localhost:8000/mystream.ogg

4.5 The administration interface

  • Available at (example):
    http://localhost:8000/admin/stats.xsl

  • Functionality:
    1. Gather statistics,
    2. move listeners from mountpoint to mountpoint,
    3. disconnect connected sources,
    4. disconnect connected listeners,
    5. and more ....

4.6 Feeding Icecast

# Capturing the WebCam and sending the stream by using ffmpeg + ffmpeg2theora + oggfwd: 
ffmpeg -f alsa -i default \ 
  -f video4linux2 -s 640x480 -r 30 -i /dev/video0 -f avi - | \ 
ffmpeg2theora -a 0 -v 5 -f avi -x 640 -y 480 -o /dev/stdout - | \ 
oggfwd localhost 8000 hackme /ejemplo.ogg 
 
# Capturing the WebCam and sending the stream with VLC: 
cvlc v4l2:///dev/video0 :input-slave=alsa:// \ 
  --sout ’#transcode{vcodec=theo,vb=800,width=640,\ 
heigh=480,acodec=vorb,ab=64,channels=2,samplerate=44100}:\ 
duplicate{dst=std{access=shout,mux=ogg,dst=source:\ 
hackme@localhost:8000/ejemplo.ogg},dst=display}’ :sout-keep

# Using netcat:

# Using wget:
